# 核心参数调优指南

本文档详细说明各参数的取值范围、默认值选择依据，以及大小变化对检索和生成的影响。

---

## 切片参数

### CHUNK_SIZE

- **默认值**: 600
- **取值范围**: 200 ~ 2000
- **说明**: 每个文本片段的最大字符数。切片器会在不超过此上限的前提下，按分隔符优先级尽量在自然断句处切分。

| 取值              | 影响                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------ |
| 过小（< 300）     | 片段信息碎片化，上下文不完整，LLM 难以理解语义；索引条数大幅增加，检索和存储开销上升 |
| 适中（400 ~ 800） | 每个片段包含完整语义单元，检索精度和 LLM 理解能力平衡                                |
| 过大（> 1500）    | 片段内信息过多，向量表示被稀释（语义偏移），检索时容易引入无关内容；token 消耗增加   |

> 中文一句话平均 30~50 字，一个段落约 100~300 字，600 字约覆盖 2~3 个段落，是一个较平衡的粒度。

### CHUNK_OVERLAP

- **默认值**: 80
- **取值范围**: 0 ~ CHUNK_SIZE / 3
- **说明**: 相邻片段重叠的字符数，避免关键信息恰好在切片边界被截断。

| 取值                     | 影响                                                 |
| ------------------------ | ---------------------------------------------------- |
| 0                        | 无重叠，边界处信息可能丢失（如一句话被拦腰截断）     |
| 适中（50 ~ 150）         | 边界处保留足够上下文，关键信息不会因切分而丢失       |
| 过大（> CHUNK_SIZE / 3） | 重叠过多导致索引冗余，相似片段数量膨胀，检索结果重复 |

> 经验值：`CHUNK_OVERLAP ≈ CHUNK_SIZE × 10% ~ 15%`。600 × 10% = 60，取 80 留有一定余量。

## 检索参数

### SEARCH_TOP_K

- **默认值**: 10
- **取值范围**: 3 ~ 30
- **说明**: 最终返回给 LLM 的检索片段数量。内部实际从 ES 每路取 `top_k × 3` 条候选，经 RRF 融合后再截取 top_k。

| 取值           | 影响                                                                       |
| -------------- | -------------------------------------------------------------------------- |
| 过小（3 ~ 5）  | 混合检索时 BM25 精确匹配的结果容易被 KNN 高分噪声挤出；LLM 上下文信息不足  |
| 适中（7 ~ 15） | 融合排名有足够空间让 BM25 结果进入；LLM 获得充分的参考信息                 |
| 过大（> 20）   | 引入过多低相关度片段，LLM 可能被噪声干扰导致回答偏题；token 消耗和延迟增加 |

> 混合检索场景建议 >= 7。纯向量检索可以适当减小到 3~5。

### SEARCH_NUM_CANDIDATES

- **默认值**: 200
- **取值范围**: SEARCH_TOP_K ~ 500
- **说明**: KNN 检索的粗筛候选数（ES `num_candidates` 参数）。HNSW 算法先从全量向量中粗筛出 num_candidates 个近邻，再精确排序取 top k。

| 取值              | 影响                               |
| ----------------- | ---------------------------------- |
| 过小（< 50）      | 粗筛不足，漏掉相关文档，召回率下降 |
| 适中（100 ~ 300） | 召回率和性能的平衡点               |
| 过大（> 500）     | 检索延迟增加，但召回率提升边际递减 |

> 经验值：`num_candidates >= top_k × 5`。当数据量大（> 10 万条）时适当增大。

## RRF 融合参数

### RRF_RANK_CONSTANT (k)

- **默认值**: 60
- **取值范围**: 1 ~ 200
- **说明**: RRF 公式 `1 / (k + rank)` 中的常数 k。k 越大，排名靠前的文档优势越小，融合结果更"民主"。

| 取值             | 影响                                                                        |
| ---------------- | --------------------------------------------------------------------------- |
| 过小（1 ~ 10）   | 排名第 1 的文档获得压倒性优势，后续排名贡献极小，融合退化为"谁排第一就选谁" |
| 适中（30 ~ 100） | 前排文档有适度优势，但不会被完全独占；两路检索的结果都能影响最终排名        |
| 过大（> 200）    | 所有排名的分数趋于均等，融合失去区分度，排序接近随机                        |

> 学术论文和 Elasticsearch 默认值均为 60，是经过大量实验验证的平衡点。

### KNN_WEIGHT / BM25_WEIGHT

- **默认值**: 0.3 / 0.7
- **约束**: KNN_WEIGHT + BM25_WEIGHT = 1.0
- **说明**: 两路检索在 RRF 融合中的权重占比。

| 权重组合  | 适用场景                                                        |
| --------- | --------------------------------------------------------------- |
| 0.5 / 0.5 | 查询词与原文高度一致，向量和文本检索互补均等                    |
| 0.3 / 0.7 | 查询词可能包含近义词/同义词（中文常见），BM25 精确匹配更可靠    |
| 0.7 / 0.3 | 查询偏语义/模糊（如"贾府衰败的原因"），向量检索更能捕捉语义关联 |
| 0.0 / 1.0 | 纯 BM25，退化为关键词搜索                                       |
| 1.0 / 0.0 | 纯向量检索，退化为语义搜索                                      |

> **调优经验**：中文嵌入模型（如 BGE）对近义词区分不足（如"温柔"vs"风流"），向量检索可能漏掉原文精确匹配的片段。此时 BM25 权重需 >= 0.7 才能保证精确匹配结果不被 KNN 噪声挤出。

## 向量维度

### VECTOR_DIMS

- **默认值**: 1024
- **取值范围**: 由嵌入模型决定，不可随意修改
- **说明**: 嵌入模型输出的向量维度。BGE-large-zh-v1.5 输出 1024 维，BGE-base-zh 输出 768 维。

> 此值必须与嵌入模型匹配，修改后需 `--rebuild` 重建索引，否则 ES 写入和检索都会报错。

## 切片分隔符

### CHUNK_SEPARATORS

- **默认值**: `["\n\n", "\n", "。", "！", "？", "；", "，", " "]`
- **说明**: 切片器的分隔符优先级列表。切片器按列表顺序尝试分隔，优先在高级别分隔符处断开。

| 优先级 | 分隔符 | 语义               |
| ------ | ------ | ------------------ |
| 1      | `\n\n` | 段落边界（最优先） |
| 2      | `\n`   | 行边界             |
| 3      | `。`   | 句号               |
| 4      | `！`   | 感叹号             |
| 5      | `？`   | 问号               |
| 6      | `；`   | 分号               |
| 7      | `，`   | 逗号               |
| 8      | ` `    | 空格（最后手段）   |

> 中文文本不建议使用默认的英文分隔符（如 `. `），上述中文优先级列表能更好地保留句子完整性。

## 调优速查

| 现象                        | 可能原因             | 调整方向                                      |
| --------------------------- | -------------------- | --------------------------------------------- |
| 检索结果全是无关片段        | KNN 向量语义偏移     | 增大 BM25_WEIGHT；增大 SEARCH_TOP_K           |
| 检索结果相关但 LLM 回答偏题 | 噪声片段过多         | 减小 SEARCH_TOP_K；增大 BM25_WEIGHT           |
| 原文有的内容检索不到        | 切片粒度太粗或太细   | 调整 CHUNK_SIZE；增大 SEARCH_NUM_CANDIDATES   |
| 答案被截断/不完整           | 切片把关键信息截断了 | 增大 CHUNK_OVERLAP                            |
| 检索速度慢                  | num_candidates 过大  | 减小 SEARCH_NUM_CANDIDATES；减小 SEARCH_TOP_K |
